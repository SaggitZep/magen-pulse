# Magen Pulse V2.0

הגרסה מחליפה את מנגנון הניקוד הקודם.

## מה השתנה
- אין ציון בסיס ידני.
- אין תרומות סופיות שמוזנות ידנית.
- כל אות כולל: עוצמה, אמינות, עדכניות, השפעה מרבית ורלוונטיות לכל חלון.
- המנוע משתמש ב־Noisy‑OR כדי לצבור ראיות בלי סכימה גסה.
- מניעת ספירה כפולה באמצעות dependency_group.
- אותות מפחיתי סיכון מפחיתים באופן מוגבל בלבד.
- שלושה חלונות: 60 דקות, 6 שעות, 24 שעות.
- לכל ציון מוצג טווח אי־ודאות ורמת ביטחון.
- רענון חישובי כל 10 דקות.

## מגבלה חשובה
הסקריפט מחשב מחדש את הציון, אך אינו אוסף חדשות בעצמו. כדי להפוך את המערכת לחיה באמת יש לחבר קולקטורים למקורות ציבוריים, לבצע דה־דופליקציה ולאמת ממצאים. עד אז הנתונים בקובץ risk-data.json הם תמונת מצב מאומתת ידנית.

## פריסה
יש להחליף את כל קבצי המאגר בקבצי החבילה, להשאיר את תיקיית `.git`, לבצע Commit ואז Push.
